Your own development platform.
One command.

Git hosting, CI/CD, object storage, database, and Kubernetes dashboard. Self-hosted. Open source. Yours.

curl -fsSL https://open-platform.sh/install | bash

What you get

Forgejo

Git hosting + identity provider. Repos, issues, pull requests, packages, and OAuth2/OIDC for all services.

Woodpecker CI

Continuous integration and deployment. Push to deploy. Preview environments for every PR.

PostgreSQL

Managed database via CloudNativePG. Automatic provisioning per app. Schemas applied in CI.

MinIO

S3-compatible object storage. File uploads, assets, backups. One bucket per app, auto-provisioned.

Kubernetes

k3s under the hood. Lightweight, production-grade. Headlamp dashboard for visibility.

Flux GitOps

Platform manages itself. Push config changes to Git, Flux reconciles automatically.

Quick Start

1

Install

curl -fsSL https://open-platform.sh/install | bash

Installs k3s, Helm, and deploys the full platform. Takes about 5 minutes.

2

Set DNS

Point *.your-domain.com to your server's IP address. A wildcard DNS record covers all services.

3

Sign in

Visit forgejo.your-domain.com and sign in with your admin credentials. Create repos, push code, deploy apps.

How it works

Browser --> Traefik (ingress)
              |-- forgejo.* --> Forgejo (git, identity)
              |-- ci.*      --> Woodpecker (CI/CD)
              |-- headlamp.* -> Headlamp (dashboard)
              |-- minio.*   --> MinIO (S3 storage)
              |-- your-app.* -> Your Apps

Forgejo --> PostgreSQL (managed by CNPG)
Woodpecker --> Forgejo (OAuth2 + webhooks)
Flux --> Forgejo (GitOps reconciliation)

Requirements

  • Linux server (Ubuntu, Debian, RHEL, NixOS, etc.)
  • 2+ CPU cores
  • 4+ GB RAM
  • 20+ GB disk
  • Domain with wildcard DNS (*.your-domain.com)
  • Ports 80 and 443 accessible (for Let's Encrypt and HTTPS)

FAQ

How do I update?

cd /opt/open-platform && git pull && make deploy

How do I back up my data?

Forgejo data, PostgreSQL, and MinIO all use persistent volumes. Back up the k3s data directory at /var/lib/rancher/k3s or use your preferred volume snapshot tool.

Can I add more nodes?

Yes. k3s supports multi-node clusters. Join additional nodes with k3s agent. Multi-node documentation coming soon.

What about air-gapped environments?

Set tls.mode: selfsigned in open-platform.yaml. The installer generates a self-signed CA and wildcard certificate automatically.

Is it production-ready?

It runs real workloads on k3s, the same distribution used by thousands of production clusters. Single-node mode is great for small teams. HA mode is on the roadmap.
