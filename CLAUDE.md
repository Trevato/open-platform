# Open Platform — Agent Reference

Self-hosted developer platform running on k3s. All services authenticate through Forgejo as the OIDC/OAuth2 identity provider. The platform is self-seeding and self-managing via Flux GitOps. Domain, TLS mode, and optional Cloudflare Tunnel are configurable via `open-platform.yaml`.

## Architecture

```
                    ┌─── Cloudflare Tunnel (optional) ───► *.open-platform.sh
                    │
Browser ──► Traefik (ingress, hostNetwork) ──┬── forgejo ────► Forgejo (git, packages, OIDC provider)
              *.{PLATFORM_DOMAIN}             ├── ci ─────────► Woodpecker (CI/CD)
                                              ├── minio ──────► MinIO Console
                                              ├── s3 ─────────► MinIO S3 API
                                              ├── oauth2 ─────► OAuth2-Proxy (preview auth)
                                              ├── mail ───────► Mailpit (SMTP catch-all, dev mode)
                                              ├── meet ───────► Jitsi Meet (video conferencing)
                                              ├── chat ───────► Zulip (team messaging)
                                              ├── console ────► Console (platform control panel)
                                              ├── api ────────► Platform API (REST + MCP)
                                              └── <app> ──────► Apps (from template)

Console ──► op-api (REST) ──► Forgejo API + K8s API
op-api ──► MCP Server (~43 tools across 10 categories)

Forgejo ──► PostgreSQL (CNPG cluster, postgres namespace)
Woodpecker ──► Forgejo (SCM integration, OAuth2)

Flux (flux-system) ──► system/open-platform on Forgejo ──► reconciles all platform services
```

### Configuration System

Single source of truth: `open-platform.yaml` (used as Helm values file) → `helm upgrade --install` → Flux reconciles.

- **`open-platform.yaml`** — User config (domain, admin, TLS mode, Cloudflare). See `open-platform.yaml.example`.
- **TLS modes**: `selfsigned` (auto-generated CA), `letsencrypt` (cert-manager + HTTP-01), `cloudflare` (tunnel at edge)
- **`service_prefix`** — Optional prefix prepended to all service subdomains (e.g., `myteam-` → `myteam-forgejo.open-platform.sh`). Empty by default.
- **Secrets** — Managed by Helm `lookup` function. Auto-generated on first install, persisted in-cluster, preserved across upgrades.

### Domain Strategy

The domain is set in `open-platform.yaml` (e.g., `domain: dev.test` or `domain: open-platform.sh`). All services use `*.{PLATFORM_DOMAIN}`.

Forgejo's `ROOT_URL`, Woodpecker's `WOODPECKER_HOST`, and app `BETTER_AUTH_URL` all use `https://<service>.{PLATFORM_DOMAIN}`. In-cluster service-to-service communication uses Kubernetes DNS (e.g., `forgejo-http.forgejo.svc.cluster.local`).

In multi-tenant mode (`service_prefix` set), all service URLs become `{prefix}{service}.{PLATFORM_DOMAIN}`. The prefix is propagated to Woodpecker org secrets for CI pipelines.

### Cloudflare Tunnel (optional)

Enabled when `cloudflare` section is present in `open-platform.yaml`.

- **Namespace**: `cloudflare`
- **Routes**: `*.{PLATFORM_DOMAIN}` → `http://traefik.kube-system.svc.cluster.local`
- **Credentials**: `cloudflared-credentials` secret (auto-generated by Helm on first install)

### Preview Environments

PR previews deploy at `pr-N-<app>.open-platform.sh`, protected by OAuth2-Proxy (Forgejo login required):

- **Namespace**: `op-{org}-{repo}-pr-{number}` (e.g., `op-system-social-pr-1`)
- **Domain**: `pr-{number}-{repo}.open-platform.sh` (e.g., `pr-1-social.open-platform.sh`)
- **Auth**: Traefik ForwardAuth middleware → OAuth2-Proxy → Forgejo OIDC
- **Isolation**: Separate DB, S3 bucket, and namespace per PR
- **Seed data**: Applied via `_seed_marker` pattern — checks for marker row before seeding to avoid duplicates on re-runs
- **Cleanup**: `preview-cleanup.yml` deletes namespace (cascades all resources), DB, and bucket on PR close

### Two Deployment Models

- **Helm chart + Flux** manages the platform. The Helm chart installs Flux CRDs and bootstrap resources. Flux then reconciles HelmReleases + manifests from `system/open-platform` on Forgejo. `platform/` directory is the source of truth for Flux-managed resources.
- **Woodpecker CI** deploys apps. App template includes 4 workflows: deploy (main branch), preview (PR — isolated DB/bucket per PR), preview-cleanup (PR close), and reset (manual data wipe).

| ------------ | ------------------- | -------------------------------------------------- | -------------------------------------------------------------------------------------- |

| Woodpecker   | `woodpecker`        | `ci.{PLATFORM_DOMAIN}`                             | CI/CD. K8s-native backend, Kaniko image builds, auto-activated repos.                  |

| OAuth2-Proxy | `oauth2-proxy`      | `oauth2.{PLATFORM_DOMAIN}`                         | Protects PR preview environments via Traefik ForwardAuth.                              |

| Jitsi Meet   | `jitsi`             | `meet.{PLATFORM_DOMAIN}`                           | Video conferencing. JWT auth via OIDC adapter. Enabled by default.                     |

| Platform API | `op-system-op-api`  | `api.{PLATFORM_DOMAIN}`                            | REST API + MCP server. Forgejo PAT auth. ~43 MCP tools across 10 categories.           |
| Flux         | `flux-system`       | n/a                                                | GitOps — watches `system/open-platform`, reconciles all HelmReleases.                  |

For full config details (Helm charts, secrets, RBAC, env vars), see [docs/services.md](docs/services.md).

### Platform API (`apps/op-api/`)

- **Source**: `apps/op-api/`
- **Namespace**: `op-system-op-api`
- **Domain**: `api.{PLATFORM_DOMAIN}`
- **Runtime**: Elysia on Bun
- **Auth**: Bearer token (Forgejo PAT) validated against Forgejo API. Admin check via `system` org membership.
- **REST API**: 12 route plugins — status, users, orgs, repos, PRs, branches, files, issues, pipelines, apps, platform (admin), mcp-tools. Swagger docs at `/swagger`.
- **MCP Server**: ~43 tools across 10 categories — orgs (2), repos (5), PRs (7), issues (7), branches (3), files (3), pipelines (4), apps (6), users (1), platform (5). Streamable HTTP transport at `/mcp` via `@modelcontextprotocol/sdk`. Per-session `McpServer` instances via `createMcpServer(user)`. Session-based with 30-min TTL, stale cleanup every 5 min.
- **Database**: `platform_ledger` (shared with console) — `platform` table.
- **K8s access**: ServiceAccount `op-api` with ClusterRole for reading platform service statuses, namespaces, and deployments.
- **Key env vars**: `FORGEJO_INTERNAL_URL` (HTTP, server-to-server), `FORGEJO_URL` (HTTPS, external), `WOODPECKER_INTERNAL_URL`, `DATABASE_URL`, `PLATFORM_DOMAIN`, `SERVICE_PREFIX`.

### Console (Platform Control Panel)

- **Source**: `apps/console/`
- **Namespace**: `op-system-console`
- **Domain**: `console.{PLATFORM_DOMAIN}`
- **Runtime**: Next.js 15 (App Router)
- **Database**: `platform_ledger` on host PostgreSQL
- **Auth**: Forgejo OAuth2 via better-auth
- **Backend**: Calls op-api via internal HTTP (`http://op-api.op-system-op-api.svc:80`)
- **Role**: Platform control panel for admins. App directory (list, create, manage apps), MCP connector (connection info and config snippets for Claude Code, Cursor, VS Code, Continue.dev, Windsurf).
- **Schema**: `platform` (single-row config)
- **Pages**: Landing (`/`), dashboard (`/dashboard`), apps (`/dashboard/apps`), MCP (`/dashboard/mcp`)
- **Permission model**: Binary admin/user. Admin = Forgejo `is_admin` or `system` org member.
- **Key env vars**: `DATABASE_URL`, `BETTER_AUTH_URL`, `AUTH_FORGEJO_URL`, `AUTH_FORGEJO_INTERNAL_URL`, `OP_API_URL`, `PLATFORM_DOMAIN`.

## System Org

Created by the bootstrap Job during first deploy. Contains:

| Repo                   | Purpose                                                                                                                                                                          |
| ---------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `system/open-platform` | Platform config — Flux HelmReleases + K8s manifests. Source of truth for all platform services.                                                                                  |
| `system/template`      | App template — Next.js 15 App Router with postgres, S3/MinIO, Forgejo OAuth2 via better-auth, declarative schema, seed data, 4 Woodpecker CI workflows. Marked as template repo. |
| `system/console`       | Platform control panel — app directory, MCP connector. Deployed at `console.{PLATFORM_DOMAIN}`.                                                                                  |
| `system/op-api`        | Platform API — REST (Elysia/Bun) + MCP server for repos, pipelines, apps, platform management. Deployed at `api.{PLATFORM_DOMAIN}`.                                              |

## Deployment

### Bootstrap (first deploy)

```bash
cp open-platform.yaml.example open-platform.yaml  # set domain, admin, TLS mode
make deploy                                        # helm install → Flux takes over
```

### Steady state (after bootstrap)

Push changes to `system/open-platform` on Forgejo → Flux reconciles within ~1 minute.

### Recovery

```bash
make deploy                 # re-bootstraps everything, Flux re-adopts
```

### Day-to-Day

```bash
make deploy    # idempotent helm upgrade --install
make upgrade   # upgrade chart version
make template  # render chart locally (debug)
make lint      # validate chart
make status    # check release status
```

### Deployment Layers

Helm chart installs Flux CRDs → Flux reconciles in dependency order: infra-controllers → infra-configs → identity → apps. Dependencies are expressed via `dependsOn` in Flux Kustomizations within `platform/`.

### App Template (`templates/app/`)

Next.js 15 App Router with PostgreSQL, S3/MinIO, Forgejo OAuth2 (better-auth), 4 Woodpecker CI workflows. Apps use `AUTH_FORGEJO_URL` for OAuth2 and `BETTER_AUTH_URL` as canonical callback URL. Includes `CLAUDE.md` (project conventions), `.claude/settings.json` (Claude Code permissions + pre-commit typecheck hook), and `PLATFORM.md` (env var reference).

## Secrets Management

Secrets are managed by the Helm chart's `lookup` function — auto-generated on first install, persisted in-cluster as Kubernetes Secrets, and preserved across `helm upgrade`. No local state files or `.env` required.

- **Infrastructure secrets** — admin credentials, DB passwords, agent secrets (created by Helm templates)
- **OAuth2 secrets** — per-service OAuth2 client IDs/secrets, created by the bootstrap Job via Forgejo API
- **Flux secrets** — git credentials for Forgejo access (created by the bootstrap Job)

See [docs/services.md](docs/services.md) for full secret inventory, env vars, and per-service configuration.

### Scripts

Remaining scripts (outside the Helm chart):

- `setup-oidc.sh` — host-level k3s OIDC config (outside Helm scope)
- `check-infra.sh` — pre-deploy infrastructure validation
- `node-join.sh`, `node-remove.sh`, `node-status.sh`, `complete-mac-join.sh` — multinode management
- `teardown.sh` — destroy/cleanup

## TLS Setup

Configured via `tls_mode` in `open-platform.yaml`:

- **`selfsigned`**: Auto-generates CA + wildcard cert at `certs/`. Private keys (`certs/*.key`) are gitignored. Server node and browsers need CA trusted.
- **`letsencrypt`**: cert-manager + HTTP-01 challenge. Requires public DNS.
- **`cloudflare`**: TLS at Cloudflare edge. Traefik receives plain HTTP from tunnel.

Implementation gotchas for Woodpecker, Forgejo, Kaniko, CNPG, better-auth, OAuth2-Proxy, and Flux. See [docs/known-issues.md](docs/known-issues.md) for the full list.

## Development Conventions

- **Package management**: bun (JavaScript/TypeScript), uv (Python)
- **YAML style**: 2-space indentation, quoted strings for values with special characters
- **Helm values**: inlined in HelmRelease resources within `platform/` (Flux-managed)
- **Platform config**: in `platform/` directory, pushed to `system/open-platform` on Forgejo
- **App templates**: in `templates/` directory, pushed to `system/template` on Forgejo
- **Domain convention**: `<service>.open-platform.sh` (configurable via `PLATFORM_DOMAIN` env var)
- **Deployment**: Helm chart bootstraps, Flux manages ongoing; Makefile provides workflow targets
- **CI/CD**: Woodpecker with `.woodpecker/` directory (deploy.yml for main, preview.yml for PRs, preview-cleanup.yml for PR close, reset.yml for manual data reset). Template changes must be synced to all apps.
